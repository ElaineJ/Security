Explaination for the difference between salted and non salted rcrack strategies:

Comparing the time taken for cracking the password with and without salt,
it can be seen that a salted password requires much longer time than that
of non salted password. 

Time taken for non salted: 108.03 s
Time taken for salted: 5.21 s

For non salted passwords, the rainbow table generated to compare with the hash password 
is the same for all the passwords. One just have to compare the generated rainbow table 
with the hashed password and retrieve the entry that has the same hash to generate the plaintext.

Having a salt, we have to compute a rainbow table for each salt that we encounter. 
In this case, a random salt is placed in each password and this will require a rainbow 
table to be computed for each password that we wish to crack. 

Adding a salt to the password, increases the length of password from 5 to 6. 
The possible number of combinations to test on increases from 36^5 to 36^6. It is also 
difficult to perform a dictionary attack on the salted passwords.
